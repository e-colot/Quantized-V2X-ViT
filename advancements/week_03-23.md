## TorchAO tests
Tried automatic quantization (INT8), leading to the same AP performances and an increased inference time (on RTX 5060Ti). This is supposedly due to
- No automatic conversion available for specific layers -> no quantization applied
- No HW accelaration on the device for INT8, making it performance draining
NVFP4 automatic conversion also tried but inadequate to model structure. Would need fine-tuning, which will be time consuming.

This led to the abandon of the TorchAO library and pushed towards a manual quantization.

## Manual quantization
The core idea is to add a yaml file (matching OpenCOOD design) that controls the quantization levels.
The objective will be to measure accurate AP performances and to then turn towards an implementation that will be able to measure inference time.
* Read "Quantizing deep convolutional networks for efficient inference: A whitepaper" in order to build a quantizer module
* Quick read of "A Method of Deep Learning Model Optimization for Image Classification on Edge Device" for the same purpose
* Implemented a mantissa/exponent based FP16 fake converter
* Tested the FP16 conversion
* Implemented and tested the BP16 conversion, same process as FP16
